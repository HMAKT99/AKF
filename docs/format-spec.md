# AKF Format Specification

Complete field reference for AKF v1.0.

## Minimal Example

```json
{"v":"1.0","claims":[{"c":"Revenue was $4.2B","t":0.98}]}
```

## Envelope Fields

### v (version) — REQUIRED
```json
{"v": "1.0", "claims": [...]}
```
Semantic version string. Current version is "1.0".

### claims — REQUIRED
Non-empty array of claim objects. Each claim must have `c` (content) and `t` (trust score).

### id
Auto-generated UUID if not provided.
```json
{"id": "akf-a1b2c3d4e5f6"}
```

### by (author)
Email address or agent identifier of the creator.
```json
{"by": "sarah@woodgrove.com"}
```

### agent
AI agent identifier, set when content is AI-generated.
```json
{"agent": "copilot-m365"}
```

### at (timestamp)
ISO-8601 creation timestamp. Auto-set to current time if omitted.
```json
{"at": "2025-07-15T09:30:00Z"}
```

### label (classification)
Security classification level. Valid values:
- `public` — No restrictions
- `internal` — Organization only
- `confidential` — Need-to-know basis
- `highly-confidential` — Restricted access
- `restricted` — Highest classification

```json
{"label": "confidential"}
```

### inherit
When `true`, any derived .akf file must maintain at least the same classification level.
```json
{"inherit": true}
```

### ext (external sharing)
Whether the unit may be shared outside the organization.
```json
{"ext": false}
```

### ttl (time to live)
Retention period in days. After this period, the knowledge unit should be reviewed or deleted.
```json
{"ttl": 365}
```

### prov (provenance)
Array of provenance hops tracing the knowledge unit's history.

### hash (integrity)
SHA-256 hash of the unit contents (excluding the hash field itself). Must be prefixed with the algorithm name.
```json
{"hash": "sha256:a3f2e1d4c5b6..."}
```

### meta (metadata)
Free-form key-value metadata.
```json
{"meta": {"department": "Finance", "project": "Q3 Review"}}
```

## Claim Fields

### c (content) — REQUIRED
The knowledge claim as a human-readable string.

### t (trust) — REQUIRED
Confidence score from 0.0 (no confidence) to 1.0 (absolute certainty).

### tier (authority tier)
Integer 1-5 indicating source authority:
- **Tier 1** (weight 1.00): Primary sources — SEC filings, official records
- **Tier 2** (weight 0.85): Authoritative secondary — analyst reports, peer-reviewed
- **Tier 3** (weight 0.70): General secondary — news, industry reports
- **Tier 4** (weight 0.50): Internal estimates — CRM data, team assessments
- **Tier 5** (weight 0.30): AI inference — model outputs, extrapolations

### ver / ver_by (verification)
```json
{"ver": true, "ver_by": "sarah@woodgrove.com"}
```

### ai (AI-generated)
Flag indicating the claim was generated by an AI model.

### risk
Human-readable risk description. Recommended for AI-generated tier 5 claims.
```json
{"risk": "AI inference — extrapolated from limited data points"}
```

### decay (half-life)
Number of days until the claim's effective trust halves. Common presets:
- realtime: 0.001 days
- daily: 1 day
- weekly: 7 days
- monthly: 30 days
- quarterly: 90 days
- annual: 365 days

### fidelity (multi-resolution)
```json
{
  "fidelity": {
    "h": "Q3 rev $4.2B +12%",
    "s": "Woodgrove Q3 revenue of $4.2B, up 12% YoY driven by cloud and AI.",
    "f": "Full 2000-token detailed analysis..."
  }
}
```

## Provenance Hop Fields

### hop
Sequential integer starting at 0 for the origin.

### do (action)
One of: `created`, `enriched`, `reviewed`, `consumed`, `transformed`

### pen (penalty)
Negative float applied during transformation. Reduces effective trust.
```json
{"pen": -0.03}
```

### adds / drops
Claim IDs added or rejected during this hop.
```json
{"adds": ["c4", "c5"], "drops": ["c3"]}
```
